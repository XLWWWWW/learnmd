# GPT-4 技术报告详解：模型更新、数据收集与训练策略

## 摘要

OpenAI 的 **GPT-4 Technical Report** 介绍了 GPT-4 的能力表现、训练框架、可预测扩展方法、安全对齐流程、评测结果和已知局限。与 GPT-1、GPT-2、GPT-3 等早期论文不同，GPT-4 技术报告并没有公开模型参数量、层数、隐藏维度、注意力头数、训练 token 数、训练硬件、训练时长、具体数据配比和完整超参数。报告明确说明，这些细节因为竞争和安全原因被保留。

因此，理解 GPT-4 技术报告时需要注意一个边界：它不是一份完整的模型复现实验说明，而是一份能力、安全和训练方法概览。报告真正公开的重点包括：

- GPT-4 从纯文本语言模型扩展为多模态模型，能够接收文本和图像输入，并输出文本。
- 基础模型仍然采用预测下一个 token 的预训练方式。
- 模型在预训练后经过 RLHF 等后训练方法进行对齐。
- OpenAI 强调了可预测扩展，即在训练大模型之前，用小模型表现预测大模型 loss 和部分能力指标。
- 安全训练明显强化，引入专家红队测试、模型辅助安全数据生成、规则奖励模型等方法。
- GPT-4 在多项考试、专业基准和安全评测上显著超过 GPT-3.5，但仍存在幻觉、推理错误、偏见、过度自信和被诱导越狱等问题。

本文围绕“模型更新、数据收集、训练策略、安全对齐、评测方式、局限性”几个方面详细解释 GPT-4 技术报告中的主要内容。

## 1. 报告背景与披露边界

### 1.1 GPT-4 技术报告是什么

GPT-4 Technical Report 是 OpenAI 对 GPT-4 的系统性技术说明。报告最早发布于 2023 年 3 月，arXiv 上后续有多个版本更新。最新版主要更新作者信息、引用和部分说明，并没有把 GPT-4 的完整训练配方公开出来。

这篇报告的目标不是让外部研究者复现 GPT-4，而是说明：

- GPT-4 相比 GPT-3.5 的能力提升在哪里。
- GPT-4 的训练大体遵循什么流程。
- OpenAI 如何评估模型能力和风险。
- GPT-4 仍有哪些已知局限。
- OpenAI 在部署前做了哪些安全缓解措施。

### 1.2 报告没有公开哪些关键细节

报告中最重要的限制是信息披露不足。OpenAI 明确没有公开以下内容：

- 模型规模，例如参数量。
- 具体架构，例如层数、宽度、注意力头数量、激活函数细节。
- 硬件信息，例如使用多少 GPU、训练集群规模、训练时长。
- 训练计算量。
- 数据集构造细节。
- 数据清洗和过滤规则。
- 训练 token 总量。
- 具体优化器、学习率、batch size、训练 schedule 等超参数。

这意味着外部只能理解 GPT-4 的技术方向，而不能仅凭报告复现 GPT-4。与 GPT-3 论文公开 1750 亿参数不同，GPT-4 报告在关键工程细节上明显更封闭。

### 1.3 为什么报告选择有限披露

OpenAI 给出的理由主要有两类：

第一是竞争原因。GPT-4 是商业化系统，完整公开训练配方会削弱模型和产品的竞争优势。

第二是安全原因。越强的模型越可能被用于网络攻击、欺骗、危险知识获取、自动化滥用等场景。公开太多模型构造和训练细节，可能降低复制强模型的门槛。

因此，GPT-4 技术报告更像是一份“能力与安全报告”，而不是传统意义上的完整机器学习论文。

## 2. 模型更新：GPT-4 相比 GPT-3.5 的主要变化

### 2.1 从纯文本模型走向多模态模型

GPT-4 的一个核心变化是多模态能力。报告将 GPT-4 定义为 large multimodal model。它可以接收：

- 文本输入。
- 图像输入。
- 文本和图像混合输入。

然后输出文本结果。

这与 GPT-3、GPT-3.5 的主要公开形态不同。GPT-3 和 GPT-3.5 主要被理解为文本语言模型，而 GPT-4 已经具备图像理解能力。例如，模型可以根据图片内容回答问题、解释图表、分析截图、理解视觉场景中的对象关系。

需要注意的是，报告发布时，GPT-4 的图像输入能力并没有像文本 API 一样全面开放。OpenAI 对视觉能力采取了更谨慎的部署方式，因为图像输入会带来额外风险，例如隐私、身份识别、医疗影像误判、图像中隐含敏感信息等。

### 2.2 输出仍然是文本

GPT-4 虽然可以接收图像，但报告中的 GPT-4 输出仍然是文本。这一点很重要：它不是一个图像生成模型，而是一个能够理解图像并用自然语言回答的模型。

可以把 GPT-4 的多模态能力理解为：

```text
文本 / 图像 / 文本+图像
        ↓
      GPT-4
        ↓
      文本输出
```

这种设计使 GPT-4 可以被用于问答、分析、写作、代码、图像说明、视觉推理等任务，但不直接生成图片。

### 2.3 更强的复杂任务处理能力

GPT-4 相比 GPT-3.5 的显著提升主要体现在复杂任务上。OpenAI 官方页面指出，在普通聊天中两者差异可能不明显，但当任务复杂度足够高时，GPT-4 更可靠、更有创造性，也更能处理细微指令。

复杂任务包括：

- 多步骤推理。
- 长上下文信息整合。
- 复杂代码生成和调试。
- 法律、医学、数学等专业考试题。
- 需要约束条件的写作。
- 多轮对话中的一致性保持。
- 对图表、图像、文本混合材料的分析。

这说明 GPT-4 的提升不是简单“说话更流畅”，而是更能在复杂约束下完成任务。

### 2.4 考试和专业任务表现提升

报告列举了 GPT-4 在多项人类考试和专业基准中的表现。例如，GPT-4 在模拟律师资格考试中达到约前 10% 的水平，而 GPT-3.5 大约处于后 10%。这类结果被用来说明 GPT-4 的知识、推理、阅读理解和任务执行能力都有明显提升。

但需要注意两点：

第一，考试成绩不等于真实专业能力。模型可以在标准化题目上表现很好，但真实专业工作还涉及责任、事实核查、上下文判断、伦理和长期经验。

第二，模型仍可能产生幻觉。GPT-4 在很多 benchmark 上超过 GPT-3.5，但这不表示它的回答总是正确。

### 2.5 上下文窗口扩展

GPT-4 发布时提供了更长的上下文版本。OpenAI 页面提到 GPT-4 支持 8K 上下文，并有受限访问的 32K 版本。

上下文窗口扩大意味着模型可以在一次输入中处理更多信息，例如：

- 更长的文档。
- 多个文件片段。
- 更长的对话历史。
- 更复杂的代码上下文。
- 更大的任务说明和约束。

不过，上下文窗口更长并不自动等于模型一定能完美利用全部内容。长上下文模型仍可能遗漏中间信息、错误聚合证据、把不相关内容混在一起，或者在长输入末尾对早期信息遗忘。

### 2.6 GPT-4 不是单纯通过 RLHF 变强

报告中一个非常关键的观点是：GPT-4 的很多能力主要来自预训练，而不是 RLHF。

RLHF 的作用更偏向行为对齐，例如：

- 更愿意遵循用户指令。
- 回答方式更像助手。
- 在危险请求上拒答。
- 降低有害输出。
- 改善回答风格和可用性。

但考试、编程、数学、专业知识等能力，主要来自基础模型在大规模数据上的预训练。报告指出，在没有特殊干预时，RLHF 甚至可能损害某些 benchmark 表现。因此不能简单理解为“GPT-4 强是因为 RLHF”，更准确地说是：

```text
预训练提供基础能力
后训练塑造助手行为和安全边界
```

## 3. 数据收集：GPT-4 用了什么数据

### 3.1 公开披露的数据来源类别

报告只公开了高层数据来源。GPT-4 基础模型使用大规模数据进行训练，数据包括：

- 公开可得数据。
- 互联网数据。
- 第三方授权数据。

OpenAI 官方页面还说明，训练数据包含多种类型的文本，例如正确和错误的解题过程、强弱推理、相互矛盾和相互一致的陈述，以及多种意识形态和观点。

这类数据设计的意义是让模型接触到丰富的语言、知识、任务格式和推理样式。模型通过预测下一个 token 学习：

- 事实知识。
- 语言结构。
- 文体风格。
- 代码模式。
- 数学和逻辑表达。
- 问答格式。
- 论证结构。
- 对话模式。

### 3.2 没有公开具体数据集配方

GPT-4 报告没有公开具体数据集构成，例如：

- 互联网数据占比多少。
- 书籍数据占比多少。
- 代码数据占比多少。
- 数学数据占比多少。
- 多语言数据占比多少。
- 授权数据来自哪些供应方。
- 如何处理版权、隐私和低质量数据。
- 是否使用用户数据以及如何处理。

因此，外部不能从报告中还原 GPT-4 的数据配方。我们只能确认 GPT-4 使用了大规模混合数据，但不能准确知道每类数据的数量和比例。

### 3.3 预训练数据与能力形成

GPT-4 的基础能力来自大规模预训练。自回归语言模型的训练目标看起来很简单：给定前文，预测下一个 token。

形式上可以理解为：

```text
输入：The capital of France is
目标：Paris
```

但当数据规模、模型规模和训练计算量足够大时，这个简单目标会迫使模型学习大量隐含结构：

- 词语之间的统计关系。
- 句法和语义。
- 世界知识。
- 数学表达方式。
- 代码语法和软件工程模式。
- 常见任务的输入输出格式。
- 人类写作和推理习惯。

因此，GPT-4 的强大能力并不是因为训练目标本身复杂，而是因为模型在极大规模的数据分布中学习到了大量可迁移模式。

### 3.4 为什么数据中包含正确和错误推理都重要

OpenAI 页面提到数据中包含正确和错误的数学题解、强弱推理、矛盾和一致陈述。这一点值得特别解释。

如果模型只见过最终答案，它可能会学到表面模式，却不一定学到如何区分推理质量。如果模型在训练中见过大量正确、错误、严谨、混乱、矛盾和一致的文本，它更有机会学习到：

- 什么样的推理链条更合理。
- 哪些说法之间存在冲突。
- 答案和证据之间应该如何对应。
- 错误解法通常在哪里出问题。
- 人类如何表达不确定性和纠错。

不过，模型见过错误推理也带来风险。它可能复现错误推理，或者在不确定时编造看似合理的解释。因此后训练和评测需要继续约束模型行为。

### 3.5 授权数据的意义

第三方授权数据通常意味着 OpenAI 获得了某些非公开或受版权保护内容的使用许可。授权数据可能包含更高质量、更结构化、更专业的文本。

授权数据的潜在价值包括：

- 提高数据质量。
- 扩展专业领域覆盖。
- 降低纯互联网数据中的噪声。
- 减少版权和合规争议。

但报告没有说明授权数据具体来自哪里，也没有说明授权数据对最终能力贡献有多大。

### 3.6 后训练数据与预训练数据不同

GPT-4 的数据不能只理解为预训练语料。预训练之后，模型还需要后训练数据来塑造助手行为。

后训练数据可能包括：

- 人类示范回答。
- 人类偏好比较数据。
- 用户指令与理想回答。
- 安全拒答样本。
- 红队攻击样本。
- 专家构造的高风险场景。
- 模型辅助生成并由规则或人类筛选的数据。

这些数据的目标不是让模型记住更多事实，而是让模型学会“怎么回答”。例如：

- 普通问题要直接帮助用户。
- 危险请求要拒绝并给出安全替代。
- 模糊问题可以询问澄清。
- 不确定时要表达不确定性。
- 不应伪造引用或假装知道事实。

### 3.7 数据污染检查

由于 GPT-4 使用了大规模互联网数据，公开 benchmark 或考试题可能已经出现在训练集中。报告因此讨论了数据污染问题。

所谓数据污染，是指评测集中的题目或答案被包含在训练数据中。如果模型在训练时见过测试题，那么 benchmark 分数就不能完全代表泛化能力。

报告中采用了污染检测方法，例如通过字符串匹配检查训练数据和评测样本是否重叠，并在剔除受污染样本后重新评估。结果显示，污染对主要结论的影响有限。

这个部分说明 OpenAI 知道大规模互联网训练会带来评测污染风险，并尝试用检测方法降低误判。不过，字符串匹配并不能发现所有语义级重复，因此污染检查不能被理解为绝对保证。

## 4. 训练策略：GPT-4 是如何训练出来的

### 4.1 总体训练流程

根据报告，GPT-4 的训练可以概括为两个大阶段：

```text
大规模预训练
    ↓
基础模型 GPT-4 base
    ↓
后训练 / 对齐训练
    ↓
面向用户的 GPT-4
```

第一阶段是预训练，让模型获得通用语言、知识、推理和代码能力。

第二阶段是后训练，让模型更像一个可用、安全、遵循指令的助手。

### 4.2 预训练：预测下一个 token

GPT-4 基础模型的训练目标是 next-token prediction，也就是根据前面的 token 预测下一个 token。

例如：

```text
输入：机器学习是一种
目标：人工智能
```

或者：

```text
输入：def add(a, b):
目标：return
```

这种训练方式属于自监督学习，因为训练数据不需要人工逐条标注标签。文本本身就提供了监督信号：每一个位置的下一个 token 都可以作为目标。

这种方法的优势是：

- 可以利用海量无标注文本。
- 训练目标简单稳定。
- 容易扩展到大规模模型和数据。
- 可以自然学习语言、代码和知识。

它的局限是：

- 预测下一个 token 不等于真实理解。
- 模型可能学习到错误、偏见和有害模式。
- 模型可能为了生成流畅文本而编造事实。
- 模型本身并不知道“应该帮助用户”。

### 4.3 后训练：从语言模型到助手

基础模型只是一个强大的文本预测器。要让它成为用户可用的助手，需要后训练。

后训练的目标包括：

- 理解用户指令。
- 按照任务要求组织答案。
- 避免无关续写。
- 拒绝危险请求。
- 减少有害内容。
- 改善礼貌性、清晰度和格式。
- 在不确定时更谨慎。

GPT-4 报告说明，OpenAI 使用了类似 InstructGPT 的 RLHF 方法。InstructGPT 的典型流程是：

```text
收集人工示范数据
    ↓
监督微调 SFT
    ↓
收集人类偏好比较
    ↓
训练奖励模型
    ↓
用强化学习优化模型策略
```

GPT-4 报告没有公开完整后训练细节，但明确表明 RLHF 是 GPT-4 对齐的重要组成部分。

### 4.4 RLHF 的作用

RLHF 的核心思想是：人类很难为所有问题写出唯一标准答案，但比较两个回答哪个更好通常更容易。

基本流程可以理解为：

```text
用户问题
    ↓
模型生成多个回答
    ↓
人类标注员比较回答优劣
    ↓
训练奖励模型预测人类偏好
    ↓
用奖励模型指导语言模型优化
```

RLHF 主要解决的问题不是“让模型知道更多知识”，而是“让模型行为更符合人类偏好”。例如：

- 答案更直接。
- 更少跑题。
- 更少重复。
- 更愿意承认不确定。
- 对危险请求更谨慎。
- 对正常请求不过度拒绝。

### 4.5 RLHF 可能带来的副作用

报告中一个容易被忽视的点是：RLHF 并不总是提升所有能力指标。

原因在于，RLHF 优化的是人类偏好奖励，而不是所有 benchmark 的准确率。如果奖励模型偏好某些回答风格，模型可能变得：

- 更谨慎，但有时过度拒答。
- 更流畅，但不一定更准确。
- 更迎合用户，但可能产生迎合性错误。
- 更像助手，但在某些考试任务上表现下降。

OpenAI 页面提到，如果不加控制，RLHF 可能损害模型在部分 benchmark 上的表现。因此，后训练需要在 helpfulness、harmlessness、honesty 和 benchmark 能力之间做平衡。

### 4.6 可预测扩展：训练前预测大模型表现

GPT-4 报告非常强调 predictable scaling，也就是可预测扩展。

训练 GPT-4 这类模型成本极高，不可能像小模型一样不断试错。OpenAI 因此建立了一套方法：先训练许多更小规模的模型，根据它们的表现外推 GPT-4 的最终表现。

报告中提到两个例子：

- 用远小于 GPT-4 的模型预测最终训练 loss。
- 用较小模型预测 GPT-4 在 HumanEval 代码任务上的表现。

这说明 GPT-4 项目的关键工程能力之一，不只是把模型做大，而是让大规模训练结果在训练前可估计。

可预测扩展的价值包括：

- 降低大模型训练失败风险。
- 帮助选择训练配置。
- 提前估计模型能力。
- 减少昂贵的模型级调参。
- 让训练基础设施更可靠。

### 4.7 为什么可预测扩展很重要

大模型训练最大的风险之一是不可逆成本。训练到 GPT-4 级别需要大量算力、电力、工程资源和时间。如果训练到最后才发现 loss 异常、数据配比错误、能力不达预期，损失巨大。

可预测扩展相当于在正式训练前建立一条“经验曲线”。如果小模型的 loss 和能力指标符合预期，那么大模型训练更可能成功。如果小模型曲线异常，团队可以在低成本阶段修正问题。

这也是 GPT-4 报告中最有工程价值的部分之一：真正的大模型能力不是单靠单次训练获得，而是依赖可复用、可预测、可监控的训练系统。

### 4.8 与 Azure 超级计算基础设施的关系

OpenAI 官方页面提到，他们与 Microsoft Azure 共同设计超级计算基础设施，并重建深度学习栈。GPT-3.5 被视为一次系统测试，OpenAI 通过它发现并修复训练系统中的问题，再用于 GPT-4。

这说明 GPT-4 的训练策略不仅包括算法，也包括系统工程：

- 大规模分布式训练。
- 数据管线稳定性。
- 训练故障恢复。
- loss 监控。
- 硬件利用率优化。
- 大规模模型部署。
- 训练和推理成本控制。

在 GPT-4 级别，模型能力来自算法、数据、算力和工程系统的共同作用。

## 5. 安全对齐：GPT-4 如何降低风险

### 5.1 为什么 GPT-4 需要更强安全训练

GPT-4 的能力越强，潜在风险也越高。风险包括：

- 生成违法或危险操作步骤。
- 帮助网络攻击。
- 生成欺骗性内容。
- 帮助规避安全系统。
- 生成有害医疗、法律、金融建议。
- 放大偏见和歧视。
- 编造事实但表达得很自信。
- 被恶意 prompt 诱导绕过限制。

因此，GPT-4 的安全工作不只是内容过滤，而是训练流程、评测流程、红队测试和部署策略的组合。

### 5.2 专家红队测试

OpenAI 在部署 GPT-4 前邀请了 50 多位外部专家进行对抗测试。专家领域包括：

- AI 对齐。
- 网络安全。
- 生物风险。
- 国际安全。
- 信任与安全。
- 模型滥用。

红队测试的目标是主动寻找模型失败方式，而不是只在普通问题上评测模型。专家会尝试诱导模型输出危险、错误、违法、欺骗或越界内容。

这些发现会反馈到训练和安全缓解中。例如，如果模型在某类化学合成、生物风险、网络攻击问题上容易给出危险细节，OpenAI 可以收集相关样本，改进拒答和安全替代回答。

### 5.3 模型辅助安全数据生成

GPT-4 报告提到一种重要方法：使用模型本身辅助安全训练。

简单理解，OpenAI 不只依赖人工手写所有安全样本，而是用 GPT-4 作为分类器或辅助工具，对数据进行判断和扩展。例如：

- 判断某个请求是否属于危险类别。
- 判断模型回答是否违反安全政策。
- 帮助生成更多边界案例。
- 为训练过程提供规则化奖励信号。

这种方法的优势是可扩展。人工专家数量有限，而高风险场景种类很多。模型辅助方法可以扩大覆盖面，再由人工和规则进行把关。

### 5.4 Rule-Based Reward Models

报告中提到 rule-based reward models，简称 RBRMs。它们不是传统意义上完全从偏好数据学习出来的奖励模型，而是把安全规则转化为训练奖励信号。

可以理解为：

```text
输入：用户请求 + 模型回答
    ↓
安全规则 / 分类器判断
    ↓
给回答一个奖励或惩罚
```

RBRMs 的目标包括两个方向：

第一，该拒绝时拒绝。例如用户请求危险武器制造、恶意代码、违法操作，模型应该拒绝提供可执行细节。

第二，不该拒绝时不要拒绝。例如用户正常询问网络安全防护、化学安全教育、历史知识、医疗常识时，模型不应因为关键词敏感就机械拒答。

这两个方向同样重要。如果模型只追求安全，可能变成过度拒答；如果只追求有帮助，可能输出危险内容。RBRMs 试图让模型学会更细粒度的边界。

### 5.5 安全效果指标

报告给出了一些安全改进数据。例如，GPT-4 相比 GPT-3.5：

- 对不允许内容请求的响应倾向明显降低。
- 对敏感请求按政策处理的比例提高。
- 在 RealToxicityPrompts 等评测上有毒输出率下降。

这些结果说明后训练和安全奖励确实改变了模型行为。

但这类指标不能理解为风险完全消失。大语言模型仍可能：

- 被精心设计的提示词诱导。
- 在边界场景中判断错误。
- 对复杂高风险问题给出不完整但仍危险的信息。
- 对正常请求过度拒绝。
- 生成听起来安全但实际错误的建议。

### 5.6 安全对齐与实用性的平衡

GPT-4 的安全训练面临一个根本矛盾：模型既要 helpful，又要 harmless。

如果过度追求 helpful，模型可能对危险请求也提供详细帮助。

如果过度追求 harmless，模型可能对正常请求大量拒绝，降低可用性。

因此，安全对齐的难点不是简单让模型多说“不”，而是让模型区分：

- 合法教育问题和危险操作请求。
- 防御性网络安全和攻击性网络滥用。
- 医学科普和具体诊断治疗。
- 文学创作和现实伤害指导。
- 历史分析和仇恨煽动。

GPT-4 报告中的专家红队、规则奖励模型和后训练流程，本质上都是为了改善这种边界判断。

## 6. 评测策略：GPT-4 如何被评估

### 6.1 标准考试评测

GPT-4 报告使用了大量人类考试来评估能力，包括法律、数学、阅读、科学、编程等领域。这样做的优点是考试任务更接近人类熟悉的能力尺度，便于理解模型水平。

例如，模拟律师资格考试成绩被用来说明 GPT-4 相比 GPT-3.5 的专业推理和阅读理解能力提升。

但考试评测也有局限：

- 题目可能被训练数据污染。
- 多选题不能完全代表真实工作能力。
- 模型可能擅长答题但不擅长长期任务执行。
- 考试成绩不能直接说明模型具备责任能力。

### 6.2 机器学习 benchmark

报告还使用传统 NLP 和机器学习 benchmark，例如语言理解、推理、问答、数学、代码等任务。这些 benchmark 可以更系统地比较不同模型。

GPT-4 在很多 benchmark 上超过 GPT-3.5 和早期模型，说明它的通用能力更强。

但 benchmark 的问题是容易饱和，也容易被训练数据覆盖。随着模型越来越强，传统 benchmark 可能越来越难区分真实能力差异。

### 6.3 多语言评测

GPT-4 报告展示了多语言能力，说明 GPT-4 不只在英语上表现强。它在多个语言的 MMLU 翻译版本上也取得较高成绩。

这说明大规模多语言数据和跨语言迁移让模型具备较强的非英语能力。

不过，多语言能力并不均衡。高资源语言通常表现更好，低资源语言、方言、混合语言、专业语境仍可能存在明显问题。

### 6.4 代码评测

HumanEval 是报告中提到的重要代码评测之一。OpenAI 还用小模型外推 GPT-4 在 HumanEval 上的表现，用来展示可预测扩展能力。

代码能力的提升来自多个因素：

- 训练数据中包含大量代码。
- 模型规模更大。
- 长上下文更适合理解代码片段。
- 后训练让模型更会按指令生成代码。
- 对齐后模型更适合解释、调试和重构。

但代码评测也不能代表完整软件工程能力。真实开发还需要理解项目结构、依赖、测试、部署、安全、性能和长期维护。

### 6.5 Evals 框架

OpenAI 同期发布了 Evals 框架，用于系统评估模型行为。Evals 的价值在于：

- 让评测可重复。
- 可以持续追踪模型版本变化。
- 可以收集外部贡献的失败案例。
- 可以把真实使用中的问题转化为测试。

这说明 GPT-4 之后，模型评测不只是一次性 benchmark，而是持续工程流程的一部分。

## 7. 已知局限：GPT-4 仍然有什么问题

### 7.1 幻觉问题仍然存在

GPT-4 比 GPT-3.5 更可靠，但仍然会幻觉。幻觉指模型生成看似合理但实际错误的信息，例如：

- 编造不存在的论文。
- 给出错误事实。
- 引用不存在的来源。
- 错误解释法律或医学问题。
- 在代码中调用不存在的 API。

幻觉来自语言模型的基本训练目标：模型优化的是生成高概率文本，而不是直接连接真实世界数据库。即使模型学到了大量事实，它也没有内建的事实验证机制。

### 7.2 推理错误

GPT-4 的推理能力明显增强，但仍会在复杂推理中出错。常见错误包括：

- 多步骤数学计算中某一步出错。
- 忽略题目约束。
- 混淆充分条件和必要条件。
- 对长文本证据整合错误。
- 在看似合理的解释中隐藏逻辑漏洞。

因此，GPT-4 适合辅助推理，但不应在高风险场景中被当作自动裁决系统。

### 7.3 偏见和有害输出

GPT-4 的安全训练降低了有害输出概率，但模型仍可能继承训练数据中的偏见。偏见可能出现在：

- 性别。
- 种族。
- 宗教。
- 国籍。
- 政治立场。
- 社会阶层。
- 职业刻板印象。

对齐训练可以减少明显有害内容，但很难完全消除深层偏见，因为偏见存在于大规模文本分布和社会语境中。

### 7.4 过度自信

GPT-4 可能以非常流畅、确定的语气输出错误答案。这对用户尤其危险，因为更强的表达能力会提升错误答案的可信度。

这也是为什么报告强调模型仍需要人工监督，尤其在医学、法律、金融、安全等高风险领域。

### 7.5 可解释性不足

GPT-4 这类大模型内部机制仍然很难解释。我们可以观察输入输出行为，也可以通过 benchmark 测试能力，但很难精确说明某个答案是如何由参数内部计算得出的。

可解释性不足带来几个问题：

- 难以完全预测失败模式。
- 难以保证模型不会在特定场景中越界。
- 难以证明模型真正理解了任务。
- 难以定位内部知识和偏见来源。

### 7.6 对提示词攻击仍然敏感

即使经过安全训练，GPT-4 仍可能受到 prompt injection、jailbreak、上下文污染等攻击影响。

例如，在工具调用、浏览网页、处理用户上传文档时，恶意文本可能试图诱导模型忽略系统指令、泄露信息或执行错误操作。

这说明大模型安全不能只依赖模型本身，还需要系统层防护，例如权限隔离、工具调用限制、输入输出审计和敏感操作确认。

## 8. GPT-4 与 GPT-3/GPT-3.5 的变化对比

| 维度 | GPT-3 / GPT-3.5 | GPT-4 技术报告中的变化 |
| --- | --- | --- |
| 输入模态 | 主要是文本 | 文本 + 图像输入，文本输出 |
| 基础训练目标 | 预测下一个 token | 仍然是预测下一个 token |
| 公开程度 | GPT-3 公开较多规模细节 | GPT-4 不公开参数量、架构、训练计算等细节 |
| 能力表现 | 已具备强 few-shot 和对话能力 | 在复杂推理、考试、代码、多语言等方面显著提升 |
| 对齐方式 | InstructGPT / ChatGPT 使用 RLHF | GPT-4 继续使用 RLHF，并加强安全对齐 |
| 安全流程 | 已有对齐和内容政策 | 更系统的专家红队、模型辅助安全训练、规则奖励模型 |
| 工程重点 | 大规模预训练和指令对齐 | 强调可预测扩展和稳定大规模训练基础设施 |
| 评测方式 | 传统 benchmark + 人类偏好 | 人类考试、专业 benchmark、安全评测、污染检查、Evals |
| 主要局限 | 幻觉、偏见、不稳定 | 仍有这些问题，但频率和严重程度有所下降 |

## 9. 如何理解 GPT-4 的“技术变更”

如果把 GPT-4 的技术变更压缩成几个核心点，可以这样理解：

第一，GPT-4 不是完全改掉 GPT 系列路线，而是在已有路线上的大规模强化。基础训练目标仍然是 next-token prediction，核心仍是大规模 Transformer 类语言模型。

第二，GPT-4 的能力提升主要来自更强的预训练模型，而不是单纯来自 RLHF。RLHF 让模型更像助手，但基础能力来自数据、规模、计算和训练系统。

第三，GPT-4 的重要工程突破是可预测扩展。OpenAI 强调可以在训练大模型之前，用小模型预测大模型表现，这降低了超大规模训练的不确定性。

第四，GPT-4 的安全训练比 GPT-3.5 更系统。红队、规则奖励模型、模型辅助分类、安全数据生成和持续评测共同构成了安全流程。

第五，GPT-4 报告本身是有限披露。它告诉我们方向和效果，但没有给出完整复现方法。
